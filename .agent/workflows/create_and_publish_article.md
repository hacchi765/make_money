# ブログ記事の手動作成・投稿ワークフロー

このワークフローは、Google Adsense審査や継続的なコンテンツ充実のために、手動でブログ記事を作成し、Firebase Hostingにデプロイするための手順です。

## 前提条件
- Hugo がインストールされていること（`hugo version` で確認）
- Firebase CLI がインストールされていること（`firebase --version` で確認）
- `.env` ファイルに `FIREBASE_TOKEN` が設定されていること

---

## 1. 記事の企画・テーマ決定

Google Adsense審査に通るための記事の条件：
- **文字数：** 1,500文字以上（理想は2,000〜3,000文字）
- **独自性：** 自分の体験・意見・分析を含む
- **読者価値：** 読者の悩みや疑問に具体的に答える
- **テーマ統一：** ブログのテーマ（副業・投資・お金）に沿った内容

### おすすめ記事テーマ例
- 副業の始め方・体験談
- 投資・資産運用の解説
- AIツールの活用法
- 節約・家計管理術
- キャリア・転職アドバイス

---

## 2. 記事ファイルの作成

`blog/content/posts/` ディレクトリに Markdown ファイルを作成します。

### ファイル名の規則
- 日本語のタイトルをそのままファイル名にしてOK
- 例：`副業で月5万円稼ぐ方法.md`

### フロントマターのテンプレート

```markdown
---
title: "記事タイトル（SEOを意識した具体的なタイトル）"
date: 2026-02-18T20:00:00+09:00
draft: false
tags: ['タグ1', 'タグ2', 'タグ3']
categories: ["Money"]
description: "記事の概要（150文字程度）。検索結果に表示されるので、キーワードを含めて魅力的に書く。"
---
```

### 記事の構成テンプレート

```markdown
# 記事タイトル（H1）

導入文：読者の悩みに共感し、この記事で何が分かるかを伝える（200〜300文字）

---

## セクション1：基礎知識・背景

### サブセクション1-1
内容...

### サブセクション1-2
内容...

---

## セクション2：具体的な方法・手順

### ステップ1：〇〇する
内容...

### ステップ2：〇〇する
内容...

---

## セクション3：よくある質問・注意点

### Q：〇〇ですか？
A：〇〇です。

---

## まとめ

記事の要点をまとめる（箇条書きが効果的）

今日からできるアクション：
1. 〇〇する
2. 〇〇する
3. 〇〇する
```

## 3. AIを使った記事作成（オプション）

ChatGPTやGeminiを使って記事の下書きを作成する場合：

### プロンプトテンプレート

```
あなたは「副業・投資・お金」の専門ライターです。
以下の条件でブログ記事を作成してください。

テーマ：[記事のテーマ]
ターゲット読者：副業や投資で稼ぎたい初心者
文字数：2,000〜3,000文字
キーワード：[メインキーワード]

構成：
1. 導入（読者の悩みに共感）
2. [テーマ]とは？（基礎知識）
3. 具体的な方法・手順（Actionable Advice）
4. よくある失敗と対策
5. まとめ（今日からできるアクション）

条件：
- Markdown形式で出力
- 見出しはH2（##）とH3（###）を使用
- 具体的な数字や事例を入れる
- 専門用語は分かりやすく説明する
- 自然な日本語で書く
```

### AIで生成した記事を使う際の注意点
- **必ずリライトする：** AIの文章をそのまま使わない
- **事実確認：** 数字・統計・固有名詞は必ず確認する
- **独自性を加える：** 自分の体験・意見・感想を追加する

## 4. Hugoでビルド

// turbo
記事ファイルを作成したら、Hugoでサイトをビルドします。

```bash
cd /Users/hachiyamasaki/Git/make_money/blog && hugo --minify
```

**確認ポイント：**
- エラーが出ていないか確認
- `Pages` の数が増えているか確認
- ローカルでプレビューする場合：`hugo server` を実行して `http://localhost:1313` で確認

---

## 5. Firebase Hostingにデプロイ

// turbo
ビルドが成功したら、Firebase Hostingにデプロイします。

```bash
cd /Users/hachiyamasaki/Git/make_money && export $(grep -v '^#' .env | head -6 | xargs) && firebase deploy --only hosting --token "$FIREBASE_TOKEN"
```

**デプロイ成功の確認：**
- `✔  Deploy complete!` が表示されること
- `Hosting URL: https://x-auto-poster-bot.web.app` でサイトを確認

---

## 6. 記事の品質チェックリスト

デプロイ前に以下を確認してください：

### コンテンツチェック
- [ ] タイトルが具体的でSEOを意識している
- [ ] 文字数が1,500文字以上ある
- [ ] 導入文で読者の悩みに共感している
- [ ] 具体的な数字や事例が含まれている
- [ ] まとめで「今日からできるアクション」を提示している
- [ ] 誤字脱字がない

### 技術チェック
- [ ] フロントマターが正しく設定されている（title, date, draft: false, tags, categories, description）
- [ ] Markdownの構文エラーがない
- [ ] Hugoビルドがエラーなく完了している

### SEOチェック
- [ ] タイトルにメインキーワードが含まれている
- [ ] descriptionが150文字程度で魅力的に書かれている
- [ ] H2/H3の見出しにキーワードが含まれている

## トラブルシューティング

### Hugoビルドエラーが出る場合
```bash
# エラーの詳細を確認
cd /Users/hachiyamasaki/Git/make_money/blog && hugo --minify --verbose
```

### Firebaseデプロイが失敗する場合
```bash
# Firebaseにログインし直す
firebase login
# または、トークンを更新する
firebase login:ci
```

### 記事がサイトに表示されない場合
- フロントマターの `draft: false` を確認する
- ファイルの文字コードがUTF-8であることを確認する
- Hugoを再ビルドしてデプロイし直す
